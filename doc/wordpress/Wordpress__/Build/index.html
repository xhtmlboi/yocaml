Description of a build rule.

Type

('a, 'b) Build.t describes a special case of a function. Indeed, it is a function of 'a -> b Effect.t to which a dependency link (Deps.t) is attached.

type ('a, 'b) t

Action on rules

val dependencies : ('a, 'b) t -> Wordpress.Deps.t

dependencies rule returns the dependencies of the rule.

val task : ('a, 'b) t -> 'a -> 'b Wordpress__.Effect.t

task rule returns the task of the rule.

Building rules

Combiners to build rules (increasingly complex, to infinity and beyond).

val watch : Wordpress.Aliases.filepath -> (unit, unit) t

watch file generates an Arrow that adds a file to the dependency list without reading it. It can be useful for making file generation dependent on other files. For example :

let track_binary_update = watch Sys.argv.(0)

Which adds the generating binary to the list of dependencies.

val create_file : Wordpress.Aliases.filepath -> (unit, string) t -> unit Wordpress__.Effect.t

create_file target build_rule executes the build_rule task if the dependencies are not up-to-date for target (or target does not exist).

val process_markdown : (string, string) t

Process a string as a Markdown document. The markdown generation uses the excellent OMD library.

val read_file_with_metadata : (module Wordpress.Metadata.PARSABLE with type t = 'a) -> Wordpress.Aliases.filepath -> (unit, 'a * string) t

Read a file and parse metadata in the header. If the metadata is invalid, the arrow will throw an error. The Arrow uses ocaml-yaml for defining metadata. See Metadata.

val apply_as_template : (module Wordpress.Metadata.INJECTABLE with type t = 'a) -> ?⁠strict:bool -> Wordpress.Aliases.filepath -> ('a * string, 'a * string) t

Applies a file as a template. (and replacing the metadata). Once the content has been transformed, the arrow returns a pair containing the metadata and the file content injected into the template. The Arrow uses ocaml-mustache as template engine.

val without_body : 'a -> 'a * string

When a template should be applied without body.

val collection : 'a list Wordpress__.Effect.t -> ('a -> (unit, 'b) t) -> ('b list -> 'c -> 'd -> 'e) -> ('c * 'd, 'e) t Wordpress__.Effect.t

Sometimes it is necessary to calculate a page according to other pages, for example to make an index of articles. With collection you can separate this procedure into 3 steps.

  • First, it executes an effect that acts on a list
  • Then it goes through the list of collected data and applies an arbitrary arrow to it.
  • Finally, it applies an aggregate function to the collected list.

For example, let's build our index by projecting the list of articles into the Articles metadata:

let index =
  let open Build in
  let* articles =
    collection
      (read_child_files "articles/" (with_extension "md"))
      (fun source ->
        track_binary_update
        >>> read_file_with_metadata (module Metadata.Article) source
        >>^ fun (x, _) -> x, article_destination source)
      (fun x meta content ->
        x
        |> Metadata.Articles.make
             ?title:(Metadata.Page.title meta)
             ?description:(Metadata.Page.description meta)
        |> Metadata.Articles.sort_articles_by_date
        |> fun x -> x, content)
  in
  create_file
    (into destination "index.html")
    (track_binary_update
    >>> read_file_with_metadata (module Metadata.Page) "index.md"
    >>> snd process_markdown
    >>> articles
    >>> apply_as_template
          (module Metadata.Articles)
          "templates/list.html"
    >>> apply_as_template
          (module Metadata.Articles)
          "templates/layout.html"
    >>^ Stdlib.snd)
;;
